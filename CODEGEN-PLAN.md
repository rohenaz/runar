## Go Codegen

### Generated output

For a stateful `Auction` contract with `bid` (state-mutating) and `close` (terminal):

```go
// Code generated by runar codegen. DO NOT EDIT.
package contracts

import (
    "math/big"
    runar "github.com/icellan/runar/packages/runar-go"
)

// AuctionArgs holds the typed constructor arguments.
type AuctionArgs struct {
    Auctioneer    *string  // PubKey (hex), nil = auto from signer
    HighestBidder *string  // PubKey (hex), nil = auto from signer
    HighestBid    *big.Int
    Deadline      *big.Int
}

// TerminalOutput specifies a recipient for terminal method calls.
type TerminalOutput struct {
    Satoshis int64
    Address  string // converted to P2PKH script; leave empty to use ScriptHex
    ScriptHex string // raw locking script hex (fallback)
}

// AuctionContract is a typed wrapper for the Auction contract.
type AuctionContract struct {
    inner *runar.RunarContract
}

func NewAuctionContract(artifact *runar.RunarArtifact, args AuctionArgs) *AuctionContract {
    return &AuctionContract{
        inner: runar.NewRunarContract(artifact, []interface{}{
            args.Auctioneer, args.HighestBidder, args.HighestBid, args.Deadline,
        }),
    }
}

func (c *AuctionContract) Connect(provider runar.Provider, signer runar.Signer) {
    c.inner.Connect(provider, signer)
}

func (c *AuctionContract) Deploy(opts *runar.DeployOptions) (string, *runar.Transaction, error) {
    return c.inner.Deploy(opts)
}

// Contract returns the underlying RunarContract for advanced usage.
func (c *AuctionContract) Contract() *runar.RunarContract {
    return c.inner
}

// Bid is a state-mutating method. Sig is auto-computed from the connected signer.
func (c *AuctionContract) Bid(
    bidder *string,
    bidAmount *big.Int,
    opts *runar.StatefulCallOptions,
) (string, *runar.Transaction, error) {
    return c.inner.Call("bid", []interface{}{nil, bidder, bidAmount}, opts)
}

// Close is a terminal method. Sig is auto-computed from the connected signer.
func (c *AuctionContract) Close(outputs []TerminalOutput) (string, *runar.Transaction, error) {
    termOutputs := resolveTerminalOutputs(outputs)
    return c.inner.Call("close", []interface{}{nil, nil}, &runar.CallOptions{
        TerminalOutputs: termOutputs,
    })
}
```

### Implementation steps

1. **New file: `packages/runar-sdk/src/codegen/gen-go.ts`**
   - Pure function: `generateGo(artifact: RunarArtifact, packageName?: string): string`
   - Reuses `classifyParams`, `getUserParams`, `isTerminalMethod` from `common.ts`
   - Type mapping: `bigint` -> `*big.Int`, `boolean` -> `bool`, `Sig` -> hidden (nil), `PubKey` -> `*string` (nil = auto), `ByteString`/`Addr`/`Ripemd160`/`Sha256`/`Point` -> `string`
   - Uses `*big.Int` for all `bigint` params (safe for 256-bit values, avoids int64 overflow)
   - Generated `resolveTerminalOutputs` helper converts `TerminalOutput` to SDK format
   - Package name defaults to `contracts`, configurable via CLI `--package` flag

2. **Update `packages/runar-sdk/src/codegen/common.ts`**
   - Add `mapTypeToGo(abiType: string): string` function
   - Add Go reserved word escaping (e.g., `type` -> `typ`, `func` -> `fn`)

3. **Update `packages/runar-sdk/src/codegen/index.ts`**
   - Export `generateGo`

4. **Update `packages/runar-sdk/src/index.ts`**
   - Export `generateGo`

5. **Update `packages/runar-cli/src/commands/codegen.ts`**
   - Support `--lang go` flag
   - Output file: `<ContractName>Contract.go`
   - Add `--package <name>` option (Go package name, default: `contracts`)

6. **Tests: `packages/runar-sdk/src/__tests__/codegen-go.test.ts`**
   - Stateless contract (P2PKH) -> all methods terminal
   - Stateful contract with terminal + state-mutating methods
   - Constructor arg typing with `*big.Int`
   - Sig params hidden, PubKey as `*string`
   - Method name collision with wrapper methods

7. **Verify generated code compiles**
   - Write a small Go test that imports the generated file and calls `go vet`

## Rust Codegen

### Generated output

```rust
// Code generated by runar codegen. DO NOT EDIT.
use runar::sdk::{
    RunarContract, Provider, Signer, CallOptions, DeployOptions,
    Transaction, RunarArtifact, SdkValue,
};

pub struct TerminalOutput {
    pub satoshis: u64,
    pub address: Option<String>,
    pub script_hex: Option<String>,
}

pub struct AuctionContract {
    inner: RunarContract,
}

impl AuctionContract {
    pub fn new(
        artifact: RunarArtifact,
        auctioneer: Option<String>,
        highest_bidder: Option<String>,
        highest_bid: i128,
        deadline: i128,
    ) -> Self {
        Self {
            inner: RunarContract::new(artifact, vec![
                SdkValue::from_opt_string(auctioneer),
                SdkValue::from_opt_string(highest_bidder),
                SdkValue::Int(highest_bid),
                SdkValue::Int(deadline),
            ]),
        }
    }

    pub fn connect(&mut self, provider: Box<dyn Provider>, signer: Box<dyn Signer>) {
        self.inner.connect(provider, signer);
    }

    pub fn contract(&self) -> &RunarContract { &self.inner }
    pub fn contract_mut(&mut self) -> &mut RunarContract { &mut self.inner }

    /// Bid is a state-mutating method. Sig is auto-computed.
    pub async fn bid(
        &mut self,
        bidder: Option<String>,
        bid_amount: i128,
        options: Option<StatefulCallOptions>,
    ) -> Result<(String, Transaction), runar::Error> {
        self.inner.call("bid", vec![
            SdkValue::Auto, bidder.into(), SdkValue::Int(bid_amount),
        ], options.map(Into::into)).await
    }

    /// Close is a terminal method. Sig is auto-computed.
    pub async fn close(
        &mut self,
        outputs: Vec<TerminalOutput>,
    ) -> Result<(String, Transaction), runar::Error> {
        let term = Self::resolve_outputs(outputs);
        self.inner.call("close", vec![SdkValue::Auto, SdkValue::Auto],
            Some(CallOptions { terminal_outputs: Some(term), ..Default::default() })
        ).await
    }
}
```

### Implementation steps

1. **New file: `packages/runar-sdk/src/codegen/gen-rust.ts`**
   - Pure function: `generateRust(artifact: RunarArtifact): string`
   - Type mapping: `bigint` -> `i128`, `boolean` -> `bool`, `Sig` -> hidden, `PubKey` -> `Option<String>`, hex types -> `String`
   - Uses `snake_case` for method names (convert from camelCase)
   - Generated methods are `async` (Rust SDK uses async)
   - Constructor uses positional args (Rust doesn't have object literals like TS/Go structs)

2. **Update `common.ts`**
   - Add `mapTypeToRust(abiType: string): string`
   - Add `toSnakeCase(name: string): string` helper

3. **CLI: support `--lang rust`**, output `<contract_name>_contract.rs` (snake_case file name)

4. **Tests: `codegen-rust.test.ts`**

### Rust SDK prerequisites

Check `packages/runar-rs/src/sdk/`:
- `RunarContract::call()` must accept `Vec<SdkValue>` with `SdkValue::Auto` for auto-compute
- `CallOptions` must have `terminal_outputs` field
- Verify async signatures match

### Complexity notes

- Rust lifetimes: the wrapper owns the `RunarContract`, so no lifetime issues
- `i128` vs `BigInt`: Rust's `i128` covers most cases but not 256-bit EC values. Document that users should use the raw `RunarContract` for those edge cases
- The `SdkValue` enum may need a `from_opt_string` helper if it doesn't exist
- Error type: use whatever the Rust SDK already defines

## Python Codegen

### Generated output

```python
# Code generated by runar codegen. DO NOT EDIT.
from __future__ import annotations
from dataclasses import dataclass
from typing import Any
from runar.sdk.contract import RunarContract
from runar.sdk.types import RunarArtifact, CallOptions, DeployOptions

@dataclass
class TerminalOutput:
    satoshis: int
    address: str | None = None
    script_hex: str | None = None

class AuctionContract:
    """Typed wrapper for the Auction contract."""

    def __init__(
        self,
        artifact: RunarArtifact,
        *,
        auctioneer: str | None = None,
        highest_bidder: str | None = None,
        highest_bid: int,
        deadline: int,
    ):
        self._inner = RunarContract(artifact, [
            auctioneer, highest_bidder, highest_bid, deadline,
        ])

    def connect(self, provider: Any, signer: Any) -> None:
        self._inner.connect(provider, signer)

    def deploy(self, options: DeployOptions | None = None) -> tuple[str, dict]:
        return self._inner.deploy(options=options)

    @property
    def contract(self) -> RunarContract:
        return self._inner

    def bid(
        self,
        bidder: str | None,
        bid_amount: int,
        options: CallOptions | None = None,
    ) -> tuple[str, dict]:
        """State-mutating method. Sig is auto-computed."""
        return self._inner.call("bid", [None, bidder, bid_amount], options=options)

    def close(self, outputs: list[TerminalOutput]) -> tuple[str, dict]:
        """Terminal method. Sig is auto-computed."""
        term = [_resolve_output(o) for o in outputs]
        return self._inner.call("close", [None, None], options=CallOptions(
            terminal_outputs=term,
        ))

def _resolve_output(o: TerminalOutput) -> dict:
    if o.script_hex:
        return {"scriptHex": o.script_hex, "satoshis": o.satoshis}
    from runar.sdk.script_utils import build_p2pkh_script
    return {"scriptHex": build_p2pkh_script(o.address), "satoshis": o.satoshis}
```

### Implementation steps

1. **New file: `packages/runar-sdk/src/codegen/gen-python.ts`**
   - Pure function: `generatePython(artifact: RunarArtifact): string`
   - Type mapping: `bigint` -> `int`, `boolean` -> `bool`, `Sig` -> hidden, `PubKey` -> `str | None`, hex types -> `str`
   - Uses `snake_case` for method names and param names (same as Python contracts)
   - Constructor uses keyword-only args (`*` separator) for named params
   - `PubKey` and `Sig` types in constructor get `= None` defaults

2. **Update `common.ts`**
   - Add `mapTypeToPython(abiType: string): string`
   - Reuse `toSnakeCase` from Rust additions

3. **CLI: support `--lang python`**, output `<contract_name>_contract.py` (snake_case file name)

4. **Tests: `codegen-python.test.ts`**

### Python SDK prerequisites

Check `packages/runar-py/runar/sdk/`:
- `RunarContract.call()` signature — must accept `list` args with `None` for auto-compute
- `CallOptions` must have `terminal_outputs` field
- Check if `build_p2pkh_script` exists in `runar.sdk.script_utils`

### Complexity notes

- Python's type hints are optional but the generated code should include them for IDE support
- `from __future__ import annotations` enables `str | None` syntax on Python 3.9+
- The Python SDK may not have async methods — check and match the calling convention
- No need for `*big.Int` equivalent — Python `int` is arbitrary precision

## Suggested order

1. **Go** — most benefit (no generics, verbose SDK calls), straightforward codegen
2. **Rust** — moderate benefit, more complex due to ownership/async patterns
3. **Python** — least benefit (dynamic typing), simplest codegen

Each language can be done independently and shipped incrementally.
